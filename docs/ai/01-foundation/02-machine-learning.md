# 2.1 Introduction

Machine Learning (ML) is a subfield of Artificial Intelligence that enables computers to learn patterns from data instead of relying on explicitly programmed rules.

Instead of writing every decision rule manually, developers provide examples, and the algorithm learns a mathematical model capable of making predictions on unseen data.

Machine Learning is the foundation of many modern AI applications, including recommendation systems, fraud detection, speech recognition, autonomous driving, and Large Language Models.

---

# 2.2 Why Machine Learning?

Traditional programming works well when the decision rules are known.

Example:

```python
if temperature > 38:
    return "Fever"
else:
    return "Normal"
```

However, many real-world problems are too complex to describe using fixed rules.

Examples include:

- Detecting spam emails
- Recognizing faces
- Predicting stock prices
- Translating languages
- Detecting fraudulent transactions
- Understanding customer sentiment

Writing millions of rules manually is impossible.

Instead, we allow the computer to discover these rules automatically from historical data.

---

# 2.3 Traditional Programming vs Machine Learning

Traditional Programming

```text
Rules
      +
Data
      │
      ▼
 Program
      │
      ▼
 Output
```

Machine Learning

```text
Data
      +
Correct Answers (Labels)
            │
            ▼
    Learning Algorithm
            │
            ▼
        Trained Model
```

Inference

```text
New Data
      │
      ▼
Trained Model
      │
      ▼
Prediction
```

The key difference is that the rules are learned automatically instead of being manually written.

# 2.5 Core Concepts

## Dataset

A dataset is a collection of examples used for training and evaluation.

Example:

| Age | Salary | Bought Product |
|------|--------|----------------|
|25|3000|No|
|35|7000|Yes|
|42|10000|Yes|

---

## Feature

A feature is an input variable that helps the model make predictions.

Examples:

- Age
- Salary
- Height
- Number of purchases
- Number of website visits

Features are usually represented as vectors.

---

## Label

A label is the correct answer associated with each training example.

Example:

```
Email
↓

Spam
```

or

```
Image
↓

Cat
```

Labels are available only during training.

---

## Model

A model is the mathematical representation learned from the training data.

Instead of storing rules directly, the model stores learned parameters.

During inference, the model uses these parameters to make predictions.

---

## Prediction

Prediction is the output generated by the trained model when new data is provided.

Example:

```
Customer Information
        │
        ▼
 Trained Model
        │
        ▼
 Purchase Probability = 92%
```

---

# 2.6 Types of Machine Learning

Machine Learning is commonly divided into three major categories.

```text
Machine Learning

├── Supervised Learning

├── Unsupervised Learning

└── Reinforcement Learning
```

---

# 2.7 Supervised Learning

Supervised Learning uses labeled data.

The algorithm learns the relationship between inputs and correct outputs.

Example:

```
House Size
↓

House Price
```

Training Data

| Size | Price |
|------|--------|
|100|150k|
|120|180k|
|200|320k|

Typical applications:

- Price prediction
- Fraud detection
- Disease diagnosis
- Email spam filtering

---

## Classification

Classification predicts discrete categories.

Examples:

- Spam / Not Spam
- Fraud / Normal
- Cat / Dog
- Positive / Negative

Output:

```
One of several classes
```

---

## Regression

Regression predicts continuous numerical values.

Examples:

- Stock price
- Temperature
- House price
- Sales forecast

Output:

```
A real number
```

---

# 2.8 Unsupervised Learning

Unsupervised Learning works without labels.

The goal is to discover hidden structures within the data.

Example:

```
Customer Data
      │
      ▼
Automatically Group Customers
```

Applications include:

- Customer segmentation
- Topic discovery
- Market basket analysis
- Anomaly detection

---

# 2.9 Reinforcement Learning

Reinforcement Learning trains an agent through interaction with an environment.

```text
Environment
      ▲
      │ Reward
      │
Agent
      │
      ▼
 Action
```

The objective is to maximize long-term rewards.

Applications:

- Robotics
- Game playing
- Autonomous driving
- Resource optimization

---

# 2.10 Training vs Inference

Training

```text
Large Dataset
      │
      ▼
Learning Algorithm
      │
      ▼
Model Parameters
```

Inference

```text
New Input
      │
      ▼
Trained Model
      │
      ▼
Prediction
```

Training is computationally expensive.

Inference is typically much faster.

Large Language Models may require weeks of training but answer questions within seconds during inference.

# 2.11 Train, Validation, and Test Sets

A dataset is usually divided into three subsets.

```text
Dataset

├── Train Set
│      Learn Parameters
│
├── Validation Set
│      Tune Hyperparameters
│
└── Test Set
       Final Evaluation
```

### Train Set

Used for learning.

### Validation Set

Used for model selection and hyperparameter tuning.

### Test Set

Used only once to estimate real-world performance.

Never train on the test set.

---

# 2.12 Overfitting and Underfitting

## Underfitting

The model is too simple.

```
Poor performance on both
training and testing data.
```

Characteristics:

- High bias
- Low accuracy
- Misses important patterns

---

## Overfitting

The model memorizes the training data instead of learning general patterns.

```
Excellent training accuracy

Poor testing accuracy
```

Characteristics:

- High variance
- Poor generalization
- Sensitive to noise

---

## Good Fit

A well-trained model generalizes to unseen data.

```
Training Accuracy ≈ Testing Accuracy
```

# 2.13 Evaluation Metrics

Different tasks require different evaluation metrics.

Classification

- Accuracy
- Precision
- Recall
- F1 Score
- ROC AUC

Regression

- Mean Absolute Error (MAE)
- Mean Squared Error (MSE)
- Root Mean Squared Error (RMSE)
- R² Score

Choosing the right metric is often more important than choosing the most complex model.

# 2.14 Feature Engineering

Feature Engineering is the process of transforming raw data into useful input features.

Example

Raw Data

```
Date: 2026-07-08
```

Engineered Features

- Day of week
- Month
- Weekend
- Holiday
- Quarter

Better features usually produce better models.

Many experienced ML engineers consider feature engineering more valuable than trying many different algorithms.

# 2.16 Machine Learning vs Deep Learning

Machine Learning

- Often requires manual feature engineering
- Works well on structured data
- Usually easier to interpret
- Smaller datasets

Deep Learning

- Learns features automatically
- Excels on images, audio, and text
- Requires larger datasets
- Requires more computational resources

Deep Learning is a subset of Machine Learning.

# Production Notes

As an AI Engineer, you rarely train Machine Learning models from scratch.

Instead, you are more likely to:

- Use pre-trained models
- Build inference services
- Create data pipelines
- Develop RAG systems
- Build AI Agents
- Integrate external AI services
- Monitor production performance

Understanding Machine Learning remains essential because it explains how modern AI systems learn from data and why they behave as they do.

---

# Best Practices

- Clearly define the business problem before selecting a model.
- Spend more time improving data quality than changing algorithms.
- Separate training, validation, and testing datasets.
- Monitor production data for distribution shifts.
- Automate retraining when appropriate.
- Evaluate models using metrics aligned with business objectives.

---

# Common Mistakes

❌ Assuming more data always produces a better model.

❌ Using the test dataset during training.

❌ Ignoring feature engineering.

❌ Optimizing only for accuracy.

❌ Deploying models without monitoring.

❌ Forgetting that data distributions change over time.
